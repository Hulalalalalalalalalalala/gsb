# Three.js 滚动驱动三维长页演示

这是一个基于 Vite + React + TypeScript + Three.js 的纵向长页应用，使用滚动进度驱动三维场景动画。

## 环境要求

- Node.js >= 16.x
- npm >= 8.x 或 yarn >= 1.22.x

## 安装与开发

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

开发服务器将在 `http://localhost:5173` 启动。

## 构建

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

## 预览构建结果

```bash
npm run preview
```

## 功能说明

### 页面结构

1. **首屏**：大标题与简短说明，风格偏深色高端落地页
2. **中间两段**：玻璃拟态卡片展示，包含结构化信息（特性列表和统计数据）
3. **末屏**：项目说明和运行指南

### 三维场景特性

- **主三维体**：二十面体，使用物理材质（金属度、粗糙度、清漆效果）
- **线框附属体**：半透明线框二十面体，随滚动变化透明度
- **粒子系统**：1500个粒子组成的点云，随滚动旋转和缩放
- **灯光系统**：两盏点光源 + 环境光，灯光强度随滚动变化
- **雾效**：场景雾效随滚动变化

### 滚动驱动动画映射

| 滚动进度 | 三维场景变化 |
|---------|-------------|
| 0% - 100% | 主物体旋转：X轴旋转 0-2π，Y轴旋转 0-1.5π |
| 0% - 100% | 主物体位移：Y轴上下正弦运动，Z轴前后余弦运动 |
| 0% - 100% | 相机环绕：半径从 8 缩小到 5，轻微环绕角度变化 |
| 0% - 100% | 粒子缩放：从 1.0 缩小到 0.5，粒子层 X 轴旋转 |
| 0% - 100% | 线框透明度：从 0.3 增加到 0.7 |
| 0% - 100% | 灯光强度：点光源1从 0.5 增加到 1.5，点光源2从 1.3 减少到 0.3 |
| 0% - 100% | 雾效：近平面从 5 增加到 10，远平面从 15 增加到 30 |

### 指针轨道控制（新增）

**实现方式**：修改相机方位（camera.position），将拖拽偏移叠加到滚动驱动的相机角度上

- **拖拽旋转**：在 WebGL 画布区域按下鼠标主键拖拽，可以绕画面中心目标点自由旋转相机
- **输入叠加**：拖拽偏移与滚动进度的相机角度叠加，两者同时工作互不干扰
- **平滑过渡**：使用缓动函数平滑拖拽偏移，避免角度跳变
- **边界限制**：相机 Y 轴位置限制在 [-5, 5] 范围内，防止过度旋转
- **松手结束**：指针移出画布后松手也能正确结束拖拽

### 射线与悬停反馈（新增）

使用 Three.js 自带的 Raycaster 实现：
- **射线检测**：每帧从相机向指针方向发射射线
- **悬停反馈**：射线命中主三维体时，通过增加 emissive（自发光）给出可见反馈
- **性能优化**：复用 Vector2 对象避免每帧无谓分配
- **状态管理**：仅在悬停状态变化时更新材质，避免频繁修改

## 逐步验证方式

1. **启动项目**：运行 `npm run dev` 并在浏览器中打开页面
2. **初始状态**：页面加载后，观察顶部的二十面体处于初始位置
3. **向下滚动**：
   - 慢慢向下滚动页面，观察主物体开始旋转和移动
   - 注意线框的透明度逐渐增加
   - 观察粒子系统的缩放和旋转变化
   - 注意灯光颜色和强度的变化
   - 观察相机的远近和视角变化
4. **滚动到底部**：三维场景达到最终状态，主物体旋转多圈，相机靠近
5. **向上滚动**：三维场景会平滑地回退到之前的状态
6. **拖拽旋转测试**：
   - 在 WebGL 画布区域按下鼠标左键并拖拽
   - 观察相机绕中心旋转，拖拽结束后相机位置平滑过渡
   - 同时滚动页面，验证拖拽与滚动可以叠加工作
   - 将指针移出画布后松手，验证拖拽正确结束
7. **悬停反馈测试**：
   - 将鼠标移动到主三维体（二十面体）上
   - 观察物体出现自发光效果（emissive）
   - 将鼠标移开，观察自发光效果消失
8. **响应式测试**：调整浏览器窗口大小，观察页面布局自适应变化
